# 什么是 JavaScript？

JavaScript：是一门跨平台、面向对象的编程语言。

单纯的说 ”编程语言“，其实是没有意义的，需要一种能够将这门编程语言解释、编译成低级语言，最终被操作系统执行才具有意义。 比如：

alert('Hello World')

在没有涉及到解释器、编译器之前，我可以把它叫做 ”PHP源代码“，也可以叫做 ”JavaScript源代码“，还可以叫做 ”Java源代码“ 等等。

编程语言，构建在解释器、编译器之上，能够被它们正确解析形成的一套规则，通过这些规则书写的文档，就被称作相应的”源代码“。 因此，上述示例中的源代码，拿到 ”PHP解释器“中执行，解释不了，拿到 ”Java编译器“ 中，编译不了，而置于 ”JavaScript引擎“ 中则能够正常被解释，因此在这个基础上，称之为 ”JavaScript源代码“ 才合理。

# JavaScript引擎

JavaScript引擎：JavaScript源代码 的解释器，能够将JavaScript源代码解析成低级语言的软件。

JavaScript引擎也不是随便的构建一些解析规则，而是根据一定的规范：ECMAScript，而实现的一个软件。

# ECMAScript

ECMAScript：它定义了”标准脚本语言“应该实现的规范。

可以由任意开放语言编写软件来实现这套规范，当实现了这套规范之后，就可以在这套软件的基础上构建出一种编程语言来方便开发，这门语言就是标准的脚本语言。而实现这套规范的软件统称叫做JavaScript引擎，并且在引擎的基础上构建了 JavaScript编程语言。

所以，严谨的说，JavaScript引擎，实现了 ECMAScript规范，而在 JavaScript引擎的基础上构建了 JavaScript编程语言。

其中，JavaScript引擎既然可以由任意开放语言实现，那么就可能存在差异，其中比较重要的：

  • V8，Chrome，Opera，和 Node，都使用了该引擎
  • SpiderMonky，FireFox 使用了该引擎

了解 ECMAScript 中的规范：

  • 语法（解析规则、关键字、流程控制、对象初始化，等等）
  • 错误处理机制
  • 数据类型
  • 基于原型的集成机制
  • 内置对象和函数
  • 严格模式
  • 模块系统
  • 基本内存模型

提示

这里可以看到，ECMAScript 仅仅规范了极其简单基础的规则，而要实现其他很多功能是完全不够的，因此，引擎一般都会通过创建新的对象的方式，扩展其他功能，因此形成了”宿主环境“。

# 宿主环境

所谓”宿主环境“，它是一套集成了JavaScript引擎，还提供了”针对计算机通用功能解决方案“的系统。

比如客户端JavaScript，浏览器就作为宿主环境，浏览器集成了引擎，并且提供了操作浏览器的一系列方案（浏览器是计算机中很常见的软件，因此这个操作浏览器的功能就很通用了）。

比如服务器端JavaScript，Node作为宿主环境，同样集成了引擎，并且提供了处理比如网络、文件系统、数据库处理等等的操作（在作为服务器端编程的时候，这些也是通用功能）。

提示

类似 jQuery，Vue等，这些虽然也提供了一系列强大的功能，但是它们不是必须的，我可以用自己写的类库，其他的框架等等，这些操作都不是计算机通用功能的解决方案，并且也没有自身集成引擎。

提示

当然，关于”宿主环境“，其实没必要严格的去定义它，只需要知道目前作为客户端的浏览器是一个宿主环境，作为服务器端的Node也是一个宿主环境即可。

# ES

ECMAScript 简称：ES。

通常说的 ES5 指 2009 年发布的规范，ES6 指 2015年发布的规范。

但是从 ES6 开始，及其后续的版本都不在使用版本号了，而是使用年份，比如 ES2015（ES6），ES2017，ES2023等等。
