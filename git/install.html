# 1. 安装

# 1.1 在 Linux 上安装

以 Ubuntu 为例：

apt-get -y install git-all

# 1.2 在 Window 上安装

下载 exe 文件，下一步即可。

# 1.3 检测安装结果

git --version

查看版本，如果能出现正确的结果，则安装成功。

# 2. 配置

# 2.1 配置文件

包含三个配置文件（不一定初始时就存在，但是设置对应级别的配置会自动生成），以 Linux 为例：

  • 项目目录下的 .git 目录下的 config 配置文件，它针对的是当前项目（仓库）的配置
  • ~/.gitconfig 或 ~/.config/git/config配置文件，它针对的是当前用户的所有仓库的配置
  • /etc/gitconfig配置文件，它针对的是所有用户，所有项目（仓库）的配置

提示

他们的优先级依次降低，即如果同时配置，最终当前项目下的配置文件生效。

注意

一般来说，大多数都是基于“用户”来配置的。

# 2.2 配置用户信息

通过 git config 命令管理配置。 配置用户信息是必须的，因为每一次提交都会使用到这些信息：

git config --global user.anme "xxx"
git config --global user.email "xxx"

提示

针对用户的配置，只需要配置一次即可；并且如果配置错误了，可以再次配置实现复写。

# 2.3 查看、检查配置信息

git config --list

它列出了当前能够找到的所有配置（默认的，或自定义的配置，以及所有配置文件中的（则有可能会看到有重复的）），或列出具体的配置项的配置：

git config xxx
git config user.name

提示

git 的命令，与 Linux 的使用习惯是差不多的，比如可以查看命令帮助 git xxx -h，比如查看命令文档 man git 或者 git xxx --help 或者 git help xxx
