查看历史，使用 git log 命令。
直接是用该命令，会按照时间倒序，列出所有的历史版本基础信息。

查看每个版本的差异

git log -p
git log --patch

在列出的历史版本中，还会显示每个版本的差异信息。

查看最近 n 个版本的历史

# n 为数字
git log -n
git log -5
git log -p -5

仅列出最近 n 个版本（时间倒序）。

限制历史记录中的基础信息

注意

注意这里仅仅是控制基础信息，版本号、作者、日期、提交描述等这些是基础信息，而使用 -p 查看差异化不算基础信息，即不受影响。

# 单行展示基础信息，仅展示版本号
git log --pretty=oneline
git log --pretty=oneline -p

# 查看简短基础信息，比单行稍多，展示版本号和作者
git log --pretty=short

# 查看默认的基础信息
git log --pretty=full
git log

# 查看全部信息：默认信息 + 提交者信息
git log --pretty=fuller

自定义显示信息

提示

还是使用 git log --pretty=-format:"xxx" 选项，上述的选项值是固定的，还可以支持自定义信息。

选项 说明
%H 完整哈希值（版本号）
%h 简写哈希值（版本号）
%T 树的完整哈希值
%t 树的简写哈希值
%P 父提交的完整哈希值
%p 父提交的简写哈希值
%an 作者名字
%ae 作者的邮件地址
%ad 作者的修订日期，此时还可以用 --date=xxx 来自定义日期格式
%ar 作者的修订日期，按“多久以前”的方式显示
%cn 提交者的名字
%ce 提交者的邮件地址
%cd 提交日期
%cr 提交日期，按“多久以前”的方式显示
%s 提交说明

注意

作者与提交者的区别：
作者：做出修改的人 提交者：执行 git commit 的人 大多数情况他们都是同一个人，但也有不同的，比如开发者只有修改权限，然后管理员统一提交的那种，在教学中很常见；或者还有一种 github 上项目，作为参与者，修改了某些内容希望提交，则发起请求，管理员通过或拉取修改交于管理员提交。

以 ASCII 字符组成的图形化显示记录

git log --graph

综合 自定义--pretty 和 --graph 实现图形化查看历史记录

当不需要查看详细的差异化数据时，比如查看历史提交，那么这两个结合更好，因为这种方式可以看到不同分值的历史提交情况。

git log --pretty=format:"%h %cn %cd %s" --graph

会得到类似如下的结果： 图示中，可以看到每个分值的情况，以及合并情况等，反正可以根据这些选项自定义显示信息。

根据作者查看它的提交历史

这个选项也很有用，在多人协作的时候，想要看看不同人员提交的版本

git log --author="xxx"
git log --author="goopuu"
