# 1. 跟踪文件

要管理文件，首先需要跟踪文件。
所谓跟踪文件，其实就是将文件纳入 git 版本控制系统中。

# 1.1 文件在仓库中，不一定就纳入了版本控制系统

仓库中的文件，不一定都是已被纳入了版本控制系统的，比如新增的文件，显然在版本控制系统中是不存在的。
不管是本地初始化的仓库，还是克隆的仓库，必须将文件纳入版本控制系统才能管理：

# 纳入具体的文件（包括路径）
git add xxx/xxx/xxx.sh

# 直接将当前目录下的所有未跟踪的文件一次性纳入
git add .

纳入版本控制系统后，文件自动变为“已暂存”状态，等待提交。

# 1.2 取消跟踪

git rm --cached xxx/xxx/xxx.sh

此时文件恢复到“未跟踪”情况，即没有被版本控制系统管理的场景，那么可以对他直接删除、或者再次跟踪。

# 2. 暂存文件

# 2.1 跟踪文件，自动暂存

如上 1.1.

# 2.2 对已修改的文件暂存

如果文件是已跟踪的，并且进行了修改，需要对其暂存，等待提交。还是使用 git add 命令实现。

# 对具体修改的文件暂存
git add xxx/xxx/xxx.sh

# 暂存当前目录下的所有已修改文件
git add .

# 2.3 取消暂存

取消暂存，文件会恢复到上一个状态：

  • 跟踪文件后，取消暂存，那么文件恢复到“未跟踪”
  • 修改暂存后，取消暂存，那么文件恢复到“已修改”
# 跟踪文件后，取消暂存
git rm --cached xxx/xxx/xxx.sh

# 修改暂存后，取消暂存
git reset HEAD xxx/xxx/xxx.sh

# 3. 查看文件状态

任何时候，都可以查看当前工作仓库的状态，它会展示（如果有）跟踪情况，以及之前介绍的三个状态的文件。

git status

# 简单信息
git status -s
git status --shrot

# 4. 查看修改

# 4.1 已修改状态的内容

未暂存，则为已修改状态，查看这个状态的内容：

git diff
git diff xxx/xxx/xxx.sh

# 4.2 已暂存状态的修改内容

git diff --staged
git diff --cached

git diff --staged xxx/xxx/xxx.sh
git diff --cached xxx/xxx/xxx.sh

# 5. 提交修改

注意

提交操作，是对“已暂存”的修改提交，将数据更新到版本控制系统的数据库中。 如果一个文件，同时存在已暂存的修改，和已修改（未暂存）的修改，提交的时候只会把“已暂存”的修改提交上去。

git commit -m '提交说明信息'

一次性暂存、提交：

git commit -a -m '提交说明信息'

注意

该命令，只能对“已跟踪”的文件，如果是未跟踪的，还是需要手动 git add xxx

# 6. 删除文件

# 6.1 未跟踪文件

上面 1.2 中，对文件取消跟踪后，文件变为“未跟踪”，或者就是一个新增的“未跟踪”文件，它并没有被版本控制系统管理，即相当于本地文件，可以直接删除。

# 6.2 已跟踪文件

已跟踪文件，要删除它，实质上是将他从“暂存区”删除，因此删除后还需要对暂存去的修改提交。

# 6.2.1 从 git 中删除，并自动删除本地该文件

# 1. 从 git 中删除，并自动删除本地该文件
git rm xxx/xxx/xxx.sh

# 2. 对暂存区提交（提交删除操作)
git commit -m 'xxx'

如果想取消删除：

# 1. 从 git 中删除，并自动删除本地该文件
git rm xxx/xxx/xxx.sh

# 2. 此时若想取消删除操作，首先，取消暂存
git reset HEAD xxx/xxx/xxx.sh

# 3. 然后，取消修改
git checkout -- xxx/xxx/xxx.sh

此时如果查看状态，将不再有该文件的删除信息，而且本地也恢复了该文件。

# 6.2.2 先手动删除本地文件，再从 git 中删除

注意

严格的说，这种操作是错误的，应该先从 git 中删除，然后决定（自动、手动）删除本地的该文件。

# 1. 手动删除本地文件
rm xxx/xxx/xxx.sh

# 2. 再从 git 中删除
git rm xxx/xxx/xxx.sh

# 3. 最后再对暂存区提交
git commit -m 'xxx'

一：如果想取消删除：

仅修改了本地，即仅删除了本地文件（git中管理的该文件还未操作）

# 1. 手动删除本地文件
rm xxx/xxx/xxx.sh

# 2. 取消修改
git checkout -- xxx/xxx/xxx.sh

此时取消了删除操作，并且取消了本地修改。

二：如果想取消删除：

不仅修改了本地，还修改了暂存区，即本地和 git 中，均删除了

# 1. 本地手动删除
rm xxx/xxx/xxx.sh

# 2. git 中也删除
git rm xxx/xxx/xxx.sh

# 3. 取消暂存
git reset HEAD xxx/xxx/xxx.sh

# 4. 取消修改
git checkout -- xxx/xxx/xxx.sh

# 6.2.3 从 git 中删除，但保留本地文件

注意

这个操作很有用的一个场景是：假设有一个缓存目录，在暂存时，忘记将该目录添加到“忽略文件”，导致这个目录下缓存文件都被 git 管理了，显然不合理，此时需要取消这个目录的 git 管理，然后再讲目录添加到忽略文件。即从 git 中删除，但本地保留。

此时，取消暂存即可：

git rm --cached xxx
git rm --staged xxx

# 6.2.4 强制删除

注意

这个操作针对的一个场景是，当删除某些文件时，这些文件已经做了修改（已修改），或者已暂存，此时如果要按照上述的操作的话，需要先取消暂存，然后取消修改，然后再删除。可以通过强制删除，则完全忽略这些文件当前已修改、已暂存的差异。

git rm -f xxx/xxx/xxx.sh

# 6.3 关于删除操作的理解

其实删除操作，无非就是两大类，三小类：

  • 两大类
    • 从本地删除
    • 从 git 中删除
  • 三小类
    • 从本地删除
    • 将“已修改”删除
    • 将“已暂存”删除

# 7. 移动文件、重命名文件

移动文件、重命名文件其实是一个意思。

# 重命名文件，只需要一条命令
git mv oldname newname

它实际上相当于三条命令：

mv oldname newname
git rm oldname
git add newname
