# 显示类型

外部显示类型： 控制盒子的布局。
内部显示类型： 控制盒子内部元素的布局。

# 块级盒子、内联盒子的表现

块级盒子：

  • 内联方向上（一般为水平方向）占满父容器在该方向上的可用空间
  • 每个盒子会换行
  • width 和 height 属性能生效
  • 内容尺寸、内边距、边框，会放大、缩小盒子，会将周围的其他元素从当前元素周围推开、靠拢，外边距有该效果

内联盒子：

  • 不会换行
  • widht 和 height 不生效
  • 内容尺寸、内边距、边框，不会放大、缩小盒子（因为上面的尺寸属性不生效），特殊的，在水平方向上，这些属性以及外边距也会将水平方向的其他元素推开（不推荐）

# 盒子尺寸

盒子的实际尺寸，包括三个部分：内容区 + 内边距 + 边框。

提示

外边距不算作盒子尺寸，它只是盒子之间的间隔。

# 标准盒模型

块级元素完全的支持了盒模型所有特性，内联元素仅支持部分。
标准盒模型包括以下特性：

  • 内容区，即视野看到的实际内容
  • 内边距，内容与边界之间的间隔
  • 边框，盒子的边界
  • 外边距，盒子与其他盒子之间的间隔

注意

通常情况下，设置的 width 和 height，如果针对的是标准盒模型，其实设置的是“内容区”的尺寸，此时若增加内边距、边界，则会造成盒子撑大的情况。

# 替代盒模型

标准盒模型的盒子实际尺寸很难控制，设置了 width 和 height 仅仅是内容区的尺寸，如果要控制盒子的尺寸，需要一些计算。
替代盒模型可以让设置的 width 和 height 就是盒子的实际尺寸，边框、内边距如果有设置，那么就是固定的值，而内容区的尺寸会自适应的放大、缩小。
一次性全局使用替代盒模型：

html {
    box-sizing: border-box;
}
*, *::before, *::after {
    box-sizing: inherit;
}

# 外边距折叠（重要）

一般说的外边距折叠，是指上、下外边距折叠，且针对的是块级元素之间。

第一种情况：正常上下布局的块级盒子
他们相邻的外边距会产生折叠，最终折叠后的外边距为较大值的那个。

<style>
    html { box-sizing: border-box; }
    *, *::before, *::after { box-sizing: inherit; }

    div { width: 300px; height: 200px; }

    .one { margin-bottom: 200px; }
    .two { margin-top: 100px;l}
</style>

<div class="one"></div>
<div class="two"></div>

最终的效果就是，这两个盒子之间的外边距被折叠为 200px。

第二种情况：嵌套块级盒子之间，没有区分上、下外边界
容器盒子，没有被渲染为块级盒子，导致容器盒子与后代块级盒子边界重叠。
这里涉及一个术语 BFC，块级格式上下文，简单的理解就是：元素实际会被渲染为块级盒子。

提示

通常情况下，块级标签，比如 div，它就会被渲染为块级盒子，但是如果没有实际的内容或高度（固定高度，最小高度，最大高度等），它在渲染的时候就像不存在一样，它的实际情况会根据内部的元素来决定，因此这个时候它被渲染的结果不是块级盒子。

这种情况下，如果对容器以及内部嵌套的后代块级盒子设置上外边距，那么他们会折叠，并溢出到容器的上外边界。比如：

<style>
    .container {
        margin-top: 100px;
    }
    .box {
        width: 200px;
        height: 100px;
        margin-top: 100px;
        background-color: skyblue;
    }
</style>
<div class="container">
    <div class="box"></div>
</div>

最终的结果就是，容器盒子 container 的上边界与嵌套盒子 box 的上边界重叠，并且他们的上外边距折叠为 200px，并且表现为了容器的上外边界，而嵌套盒子与容器的上外边距并没有。

下外边距折叠同理，同上的示例，都设置下外边距的话，也会造成容器与嵌套盒子的下边界重合，他们的下外边距折叠，并且溢出到容器的下方。

第三种情况：空的块级元素，上外边距折叠
第二种其实容器也相当于空的块级元素，只不过他们是嵌套关系；而第三种情况是空的块级元素与兄弟元素产生的“上外边距”折叠。
其实造成的原因同上，也是因为空的块级元素没有被渲染为块级盒子，导致它没有边界，于是与下方的块级盒子的上边界重叠，最终他们的上外边距产生折叠。

<style>
    .empty { margin-top: 100px; }
    .box { width: 200px; height: 100px; margin-top: 200px; }
</style>
<div class="empty"></div>
<div class="box"></div>

效果图同上。

解决外边距重叠
除了第一种情况，其他的情况都是因为盒子在渲染的时候没有被渲染为“块级盒子”，导致其边界与其他元素产生重叠，那么将这些元素设置为可以被渲染为“块级盒子”的情况即可，这就是 BFC的作用了。
BFC，有很多情况都能实现，比如浮动、定位、表格单元格、overflow、弹性元素、网格元素等。
我通常的做法是：

.handle-margin-collapsing, .handle-margin-collapsing::before, .handle-margin-collapsing::after {
    content: "";
    display: flex;
}

只需要把这个类名，加到上述第二种、第三种情况的“容器元素”上，即可以实现 BFC，以此抵消外边距折叠问题。
