# 选择器分类

# 通配选择器

使用 * 符号，匹配所有元素。

提示

指从 html根元素开始的所有元素。

# 标签选择器

直接使用“标签名称”作为选择器。

<style>
    h1  { ... }
    h2  { ... }
    p   { ... }
    div { ... }
</style>

# 类选择器

使用 class 的“任意一个属性值”作为选择器，属性值前面加一个 "."。

<style>
    .one   { ... }
    .box   { ... }
    .title { ... }
</style>

# ID选择器

使用 id 的“属性值”作为选择器，属性值前面加一个 "#"。

<style>
    #one   { ... }
    #box   { ... }
    #title { ... }
</style>

# 属性选择器

选择器 示例 描述
[attr] a[title] 匹配具有给定属性名的元素，示例为具有 title属性的 a 元素
[attr=value] a[href="https://www.baidu.com"] 给定属性键值对完全匹配的元素，示例为具有href属性且全等于https://www.baidu.com的a元素
[attr~=value] p[class~="special"] 匹配具有给定属性，并且其值完全匹配给定属性值，或者属性值中包含其中一个与给定属性值匹配的元素

# 伪类和伪元素选择器

伪类：匹配处于“特定状态”的元素。
比如，当链接处于未点击、已点击、正在点击、鼠标覆盖，比如下拉选择处于被选中状态，比如单选、多选处于被选中状态，比如输入框处于焦点状态等等。

选择器 描述
:link 匹配未点击的链接
:visited 匹配已点击的链接
:hover 匹配鼠标覆盖时的链接
:active 匹配激活（比如点击）元素
:checked 匹配被选中的单选、多选按钮
:selected 匹配被选中的下拉框
:focus 匹配获得焦点的任意表单组件
:is(selector1, selector2, ...) 匹配参数列表中的任意选择器
:not(selector1, selector2, ...) 匹配不在参数列表中的其他选择器
:first-child 第一个子元素
:last-child 匹配最后一个子元素
:nth-child(num) 匹配给定索引的子元素，索引从 1 开始，索引值可以是表达式，比如2n，2n + 1等

伪元素：看起来存在，但实际上不在文档中的元素。可以理解为“虚拟元素”

选择器 描述
::before 在给定元素之前插入“虚拟元素”
::after 在给定元素之后插入“虚拟元素”

注意

这两个伪元素有特定的用法：

div::before {
    content: '';         // 必须，内容可以是任意字符，甚至是网络图标之类的，比如 ➥  这样的符号

    // ...   其他的样式规则视情况而定
}

注意

对于链接来说，:link, :visited, :hover, :active，这四个伪类如果要使用，必须按照这个顺序书写.

# 关系操作符

操作符 描述
空格 后代关系
> 父子关系
~ 后续弟弟
+ 紧邻弟弟
, 分组

提示

关系操作符，不是选择器，它们是将单一的选择器组合形成复杂选择器集合操作符。
