# 1. 文档流

# 1.1 正常文档流

正常文档流，块级盒子上下排列独占一行，内联盒子紧密挨着。

# 1.2 脱离文档流

脱离文档流，盒子在正常文档流中表现得不存在（其他正常文档流则会占用该脱离文档流元素原来的位置），但是它会在更上层的地方展示。
以墙壁上挂的黑板为例，正常文档流就是黑板上书写的文字，脱离文档流就好像把其中的某些文件提取出来在黑板前面的空中展示。

# 2. 定位

# 2.1 静态定位

默认情况下，正常布局的盒子都是静态定位。
当然也可以显示的声明为静态定位：

.box {
    position: static;
}

# 2.2 相对定位

脱离文档流，但是原始位置保留，并且相对的是原始位置定位。 图示中可以看到几个现象（黄色部分做了相对定位）：

  • 定位的盒子由于向右移动了，导致超出了 body 宽度，下面出现了滚动条
  • 定位的盒子定位了，但是原始位置还是空出来了，说明原始位置被保留了
  • 定位的盒子覆盖了下方的盒子，说明定位盒子脱离了文档流，否则下方盒子应该是在定位盒子后面显示而不会被覆盖一部分

# 2.3 绝对定位

脱离文档流，原始位置不保留，并且相对于最近祖先被定位（非静态定位）或 body 元素（所有祖先都没有被非静态定位）的元素绝对定位。 图示中可以看到几个现象（黄色部分做了绝对定位）：

  • 定位的盒子原始位置没有保留，下方的盒子自动顶上去了
  • 定位的盒子覆盖了下方的盒子，说明定位脱离了文档流
  • 如果缩小窗口，也会发现下方会产生滚动条，因为定位超出了 body 的宽度

# 2.4 固定定位

脱离文档流，原始位置保留，并且表示为行内块，并且相对于视口定位。 图示中可以看到几个现象（黄色部分做了固定定位）：

  • 定位的盒子原始位置保留
  • 定位的盒子覆盖了下方的盒子，说明脱离了文档流
  • 定位的盒子背景色只有内容那一块，说明表现为行内块
  • 如果滚动，会发现定位盒子一直保持在那里，因为它是相对于视口定位的

# 2.5 粘性定位

当定位元素达到给定阈值时，表现为固定定位，否则表现为相对定位。

.box {
    position: sticky:
    top: 30px;
    background-color: yellow;
}

当没有滚动时，即没有达到设置的阈值（距离顶部 30px），表现为相对定位，则相对于本身距离顶部有 30px 距离： 当滚动，并且已经大于等于距离顶部 30px 距离时，它表现为固定定位，则相对于视口距离顶部有 30px 距离： 这种定位有一些很有意思的应用场景：
索引列表，比如电话簿按照存储姓名的首个字符的拼音首字母分类，向下滚动时，每滚动套一个拼英索引时，将当前拼音索引置顶展示，继续向下滚动，则下一个索引触发阈值继续置顶。
还有一个有意思的应用场景：当一块区域内容过长时，又不想做成区域滚动，而是想要页面滚动，将顶部的信息做成粘性定位，则在页面滚动时，顶部的着重内容总是在最上方显示。（虽然也可以做成固定定位，但是固定定位是相对于视口的，在不同尺寸的情况下不好处理，显然粘性定位更合理）
